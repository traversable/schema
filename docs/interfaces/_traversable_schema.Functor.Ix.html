    Functor.Ix

    A Functor that has been equipped with an index.

    In addition to map, instances of Functor.Ix also implement a method called mapWithIndex that will provide to its callback argument an additional parameter, ix.

    Note that ix does not have to be a number. If the source data type is a record, ix would be a string. For a tree, the index might be a path.

    The nice thing about separating the "index" logic into a functor is that it clearly delineates who is responsible for what:

    • When defining a Functor.Ix instance, it's your responsibility to

      1. keep track of the index
      2. make sure updates to the index are deterministic
      3. provide the index to the caller
      4. define its semantics, and communicate them clearly
